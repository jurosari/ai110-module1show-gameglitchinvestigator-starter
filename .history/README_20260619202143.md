## 🚨 The Situation

You asked an AI to build a simple "Number Guessing Game" using Streamlit.
It wrote the code, ran away, and now the game is unplayable. 

- You can't win.
- The hints lie to you.
- The secret number seems to have commitment issues.

## 🛠️ Setup

1. Install dependencies: `pip install -r requirements.txt`
2. Run the broken app: `python -m streamlit run app.py`

## 🕵️‍♂️ Your Mission

1. **Play the game.** Open the "Developer Debug Info" tab in the app to see the secret number. Try to win.
2. **Find the State Bug.** Why does the secret number change every time you click "Submit"? Ask ChatGPT: *"How do I keep a variable from resetting in Streamlit when I click a button?"*
3. **Fix the Logic.** The hints ("Higher/Lower") are wrong. Fix them.
4. **Refactor & Test.** - Move the logic into `logic_utils.py`.
   - Run `pytest` in your terminal.
   - Keep fixing until all tests pass!

## 📸 Demo Walkthrough

A step-by-step playthrough of a sample game so a reader can follow along without a video. This run uses the default **Normal** difficulty (range **1–100**, **8 attempts**).

1. **Launch the app.** Run `python -m streamlit run app.py`. The page opens on Normal difficulty. The sidebar confirms the settings: "Range: 1 to 100" and "Attempts allowed: 8". The main panel shows "Guess a number between 1 and 100. Attempts left: 8".

2. **Peek at the secret (optional).** Expand **Developer Debug Info** to see the current `Secret`, `Attempts`, `Score`, `Difficulty`, and `History`. For this walkthrough, assume the secret is **42**. Leave **Show hint** checked so the Higher/Lower feedback appears.

3. **First guess — 50.** Type `50` and click **Submit Guess 🚀**. Since 50 is above the secret, the hint reads **"📉 Go LOWER!"**. The guess is recorded in `History`, attempts tick up, and the secret stays **42** — it no longer changes on every click (the original state bug is fixed).

4. **Second guess — 25.** 25 is below the secret, so the hint now reads **"📈 Go HIGHER!"**. Because the secret is stable, each hint actually narrows the range honestly: the answer is between 26 and 49.

5. **Third guess — 42.** Enter `42` and submit. The game recognizes the win: **🎉 Correct!**, balloons drop, and a success message shows "You won! The secret was 42. Final score: …". The status flips to `won`.

6. **Game-over state holds.** After winning, the app displays "You already won. Start a new game to play again." and stops accepting guesses — no accidental extra turns. (The same locking behavior applies if you run out of attempts, which shows "Game over.")

7. **Start fresh.** Click **New Game 🔁**. `reset_game()` clears history and attempts, picks a new secret in the current difficulty's range, and sets the status back to `playing` — confirmed by "New game started."

8. **Change difficulty.** Pick **Easy** or **Hard** from the sidebar. The game auto-resets to that difficulty's range (Easy 1–20, Hard 1–50) and shows "Difficulty changed to … Game reset.", so the secret and attempt limit always match the selected difficulty.
